# Translation Integration

This project now includes a hybrid translation layer for English plus Indian languages.

## Files added

- `api/translate.php`
- `api/set_language.php`
- `config/translation.php`
- `assets/translation.js`
- `assets/translation.css`

## Language flow

- `EN` returns the original English text.
- `HI`, `TE`, `TA`, `KN`, `MR` use this order:
  1. Local dictionary
  2. Google Translate API if `GOOGLE_TRANSLATE_API_KEY` is configured
  3. LibreTranslate if `LIBRETRANSLATE_URL` is configured
  4. Original English text as fallback
- Other non-Indian languages use this order:
  1. DeepL if `DEEPL_API_KEY` is configured
  2. Google Translate API
  3. LibreTranslate
  4. Original English text as fallback

## Session handling

- Selected language is stored in `$_SESSION['lang']`
- The current HTML language attribute is generated by `translation_get_html_lang()`

## Backend configuration

Configure keys in Apache, XAMPP environment variables, or a secure server config:

- `DEEPL_API_KEY`
- `DEEPL_API_URL` optional, defaults to `https://api-free.deepl.com/v2/translate`
- `GOOGLE_TRANSLATE_API_KEY`
- `LIBRETRANSLATE_URL`
- `LIBRETRANSLATE_API_KEY` optional

Do not place API keys in JavaScript or HTML.

## How to add translation to more pages

1. Include `config/security.php` as usual. It now auto-loads `config/translation.php`.
2. Change the root tag to:

```php
<html lang="<?php echo h(translation_get_html_lang()); ?>">
```

3. Add the shared stylesheet and script:

```php
<link rel="stylesheet" href="assets/translation.css">
<?php translation_render_page_config(); ?>
<script src="assets/translation.js"></script>
```

For files inside subfolders, use `../assets/...` and `translation_render_page_config("../");`

4. Add the language dropdown where needed:

```php
<?php translation_render_language_selector(); ?>
```

5. Mark UI text for translation:

```php
<h1 data-i18n="Manage Complaints">Manage Complaints</h1>
<input data-i18n-placeholder placeholder="Area, address, or landmark">
```

6. For complaint descriptions or other user-facing content where original and translated versions should both be available:

```php
<span data-translate-content data-original-text="<?php echo h($text); ?>">
    <?php echo h($text); ?>
</span>
```

## Notes

- If translation fails, the original English text remains visible.
- The local dictionary is in `config/translation.php` and can be expanded for fixed UI phrases.
- Chart labels generated inside inline JavaScript are not translated automatically yet; translate those labels server-side if needed.
